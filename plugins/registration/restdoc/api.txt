== Registration

  Only authenticated users are allowed to access the API. Authentication is done
  by sending a Basic HTTP Authorisation header.

== Table of Contents

Contents:

== Formats

  The system REST service supports only XML and JSON input/output formats.

== Overview

  This module provides access to system registration service.

  It works on request, response princliple and it is not stateless. In general
  API post list of missing arguments to register products installed on machine.
  After sending such arguments service require others or finish successfully.


== PolicyKit Access Rights

  The following PolicyKit permission is needed to perform read/write actions:

  <i>org.opensuse.yast.modules.yapi.registration.getregistrationconfig</i>
  <i>org.opensuse.yast.modules.yapi.registration.statelessregister</i>

== Actions

Example workflow:

GET /registration

  Get List of required arguments or guid given by previous registration. Array
  of missingarguments list all arguments. Only arguments with kind "mandatory"
  and empty "value" must by filled. Others are already filled or optional.

  CURL Example: curl -u &lt;user> -H 'Content-type: text/xml' http://localhost:4984/registration.xml

  Result when some parameters required:
XmlResult: reg-parameters_required

  Result when everything registered:
XmlResult: reg-registered

POST /registration/register

  Send arguments to registration. All arguments that is mandatory and missing
  value must be send. Other can be send as well, but it is not neccessary.

  CURL example: curl -u <user> -X POST -H 'Content-type: text/xml' -d @reg-request.xml http://localhost:4984/registration/register.xml

XmlBody: reg-request

XmlResult: reg-response
