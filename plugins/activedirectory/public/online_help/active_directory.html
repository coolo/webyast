Enabling Active Directory Authentication

The Active Directory WebYaST module allows you to enable authentication of Active Directory users. Once configured, Active Directory users are available in roles management and Active Directory users are able to log into WebYaST.

You can find more information about Active Directory in SUSE Security Guide.

To enable Active Directory authentication, follow these steps:

  1. Before setting Active Directory authentication, ensure that the system time is correct, preferably by NTP synchronization. NTP client can be configured using WebYaST Time module.

    Also ensure that the hostname does not change - either configure DHCP to not change host names for a given IP address or set a fixed host name in the network settings.

  2. Check the Enable Authentication of AD users option in WebYaST Active Directory module.

  3. Enter Active Directory Domain to be used. If the machine is not a member of the given domain, you will be asked for Administrator User Name, Password and Machine Account.

  4. Click Save.
